# 🎬 Movie Recommendation System

A movie recommendation system that uses **Content-Based Filtering** approach in Streamlit UI to recommend movies based on user preferences and movie features. 

For experimental purposes, a **Collaborative Filtering** module has also been implemented using the `Surprise` but is not part of the UI.

---

## ✨ Features

- **Content-Based Filtering** — the main recommendation engine used in the UI, based on movie genres, year, and other features
- **Collaborative Filtering** — implemented for experimental/research purposes using the `Surprise` library
- **Evaluation Framework** — tools and reports to assess recommendation quality
- **Interactive UI** — Streamlit-based web application for exploring recommendations

## 🔧 Requirements

- **Python 3.11** *(recommended if using Collaborative Filtering)*

  > ⚠️ The **Collaborative Filtering** module relies on the [`Surprise`](https://surpriselib.com/) library, which has known compatibility issues with Python versions newer than 3.11. If you only intend to use **Content-Based Filtering** (the default UI mode), any newer Python version will work fine - however, in that case you should **skip installing the `surprise` package** from `requirements.txt` to avoid installation errors.

## 🚀 Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/KlipiecGD/rec_sys_project.git
cd rec_sys_project
```

### 2. Create and activate a virtual environment (Python 3.11)

```bash
python3.11 -m venv venv
source venv/bin/activate        # On Windows: venv\Scripts\activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Configure environment variables

Create a `.env` file and set TMDB API key if you want to enrich movies with additional features fetched from [The Movie Database (TMDB)](https://www.themoviedb.org/):

```bash
TMDB_API_KEY=your_tmdb_api_key_here
```

> ℹ️ The TMDB API key is **optional**. Without it, the system will use only the base MovieLens dataset features (genres, year, ratings). With it, additional movie metadata such as **cast, crew, keywords, overview, spoken languages, production countries** and more will be fetched and used to enrich recommendations. Without them you will not be able to expreriment with some of the CB models but the system will still work and provide recommendations based on the available features.

### 5. Prepare data and artifacts

> ℹ️ During dataset preparation, movies with fewer than **200 ratings** and users with fewer than **100 ratings** are excluded from the dataset. This threshold can be adjusted in [`src/config/config.yaml`](src/config/config.yaml) (`min_movie_ratings` / `min_user_ratings`).

Run the data preparation and model training scripts to generate necessary artifacts:

```bash
python -m src.data_source.movie_lens_25_downloader   # Download and prepare MovieLens 25M dataset
python -m src.data_source.tmdb_fetcher               # Fetch additional movie data from TMDB (optional)
python -m src.fitting.build_dataset                  # Build required datasets
python -m src.fitting.fit_cb_full                    # Fit content-based models and save artifacts
```

### 6. Run the application

```bash
streamlit run app.py
```

## 🤖 Recommendation Approaches

### ✅ Content-Based Filtering *(used in UI)*

The primary recommendation method. It analyzes movie attributes such as:
- **Genres**
- **Genome tags**
- **Year of release**
- Additional movie metadata and features

Optionally, if a TMDB API key is provided, the model is further enriched with:
- **Cast & Crew**
- **Movie Overview (plot)**

Models are precomputed and stored in `artifacts/full/`. For a detailed description, see [documents/description_content_based.md](documents/description_content_based.md) and [documents/features.md](documents/features.md).

### 🧪 Collaborative Filtering *(experimental only)*

Collaborative filtering has been implemented and tested for research purposes but is **not used in the UI** due to the following fundamental limitations:

- ❄️ **Cold Start Problem** — the model cannot generate recommendations for new users who have no rating history in the dataset
- 🔒 **Closed User Set** — predictions can only be made for users that exist in the training dataset (MovieLens 25M), making it unsuitable for a general-purpose UI where any user can interact with the system

It is based on user-item interaction patterns and uses the **Surprise** library.

If you want to run the CF evaluation, you can fit the CF model on the training split using:

```bash
python -m src.fitting.fit_cf_train
```

Artifacts are stored in `artifacts/cf/`.

## 📊 Evaluation

In order to evaluate the **content based models**, one need to run:

```bash
python -m src.fitting.fit_cb_train         # Fit CB models on train split only to generate evaluation artifacts
python -m src.evaluation.cb_report          # Run evaluation loop, print per-model metrics table, and save aggregated results to CSV
```

For collaborative filtering evaluation:

```bash
python -m src.fitting.fit_cf_train         # Fit CF model on train split only to generate evaluation artifacts
python -m src.evaluation.cf_report          # Run evaluation loop, print metrics, and save results to CSV
```

> ⚠️ Note: Evaluation may take a few hours to run, especially for the collaborative filtering algorithms.

Evaluation results are summarised in [documents/evaluation_results.md](documents/evaluation_results.md) and the generated CSV files in `reports/` (`evaluation_results.csv` for content-based models and `cf_evaluation_results.csv` for collaborative filtering).
